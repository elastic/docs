The elastic-cloud-enterprise-installer.sh Installation Script

NAME

bash elastic-cloud-enterprise-installer.sh — Install Elastic Cloud Enterprise

SYNOPSIS

bash elastic-cloud-enterprise-installer.sh [--coordinator-host C_HOST_IP]
[--host-storage-path PATH_NAME] [--cloud-enterprise-version VERSION_NAME]
[--debug] [--repository REPOSITORY_NAME] [--overwrite-existing-image]
[--installation-id ID] [--host-ip HOST_IP] [--availability-zone ZONE_NAME]
[--capacity MB_VALUE] [--memory-settings JVM_SETTINGS]

REQUIRED PRIVILEGES

To run this script, a user must be part of the docker group. You should not install Elastic Cloud Enterprise as the root user.

DESCRIPTION

Installs Elastic Cloud Enterprise according to the specified parameters, both to start a new Elastic Cloud Enterprise installation and to add hosts to an existing installation. Can be used to automate installation or to customize how you install Elastic Cloud Enterprise.

PARAMETERS

--coordinator-host COORDINATOR_HOST_IP
Specifies the IP address of the first host used to start a new Elastic Cloud Enterprise installation. Must be specified when installing Elastic Cloud Enterprise on additional hosts to add them to an existing installation.
--host-storage-path PATH_NAME
Specifies the host storage path used by the Elastic Cloud Enterprise installation. Defaults to /mnt/data/elastic.
--cloud-enterprise-version VERSION_NAME
Specifies the version of Elastic Cloud Enterprise to install. Defaults to :version:.
--debug
Outputs debugging information during installation. Defaults to false.
--repository REPOSITORY_NAME
Specifies the repository name where the Elastic Cloud Enterprise image is retrieved from during installation. Defaults to docker.elastic.co/cloud-enterprise.
--overwrite-existing-image
Overwrites any existing local image when retrieving the Elastic Cloud Enterprise installation image from the repository. Defaults to false.
--runner-id ID
Assigns an arbitrary ID to the host (runner) that you are installing Elastic Cloud Enterprise on. Runners provide a supervisor service on all nodes and deploy containers based on their role definitions. Using the machine name or IP address can help identify which host the runner ID refers to. Defaults to host-ip.
--host-ip HOST_IP
Specifies an IP address for the host that you are installing Elastic Cloud Enterprise on. Used for internal communication within the cluster. This must be a routable IP in your network. On AWS, the private IP of the host must be used. In VirtualBox, it is the eth0 inet address of the virtual machine. If a route to the default gateway can be determined, defaults to the IP address for the network interface.1
--availability-zone ZONE_NAME
Specifies an availability zone for the host that you are installing Elastic Cloud Enterprise on. The zone name is arbitrary but must be identical among all machines in a given zone. Zones are defined by your use case. AWS users have predefined zones. For AWS users, Elastic recommends a 2 or 3 zone deployment. Defaults to region names that begin with ece-region-.
--capacity MB_VALUE
Specifies the amount of RAM in megabytes this runner makes available for Elasticsearch clusters. Must be at least 8192 MB. For future coordinators, set this value relatively low. For future allocators, set this value to the total memory available on the machine minus some memory reserved for Elastic Cloud Enterprise and any other systems running on this host. Defaults to 85% of available RAM if not specified.
--memory-settings JVM_SETTINGS
Specifies a custom JVM setting for a service, such as heap size. Settings must be specified in JSON format. Supported services include: admin-console, zookeeper, client-forwarder, client-observer, runner, director, blueprint, constructor, allocator, services-forwarder, beats-runner, and cloud-ui.

1The default value for host-ip is passed in by the installation script. If you perform installation through some other method such as Ansible, you must specify a value for host-ip or an error is returned.

Examples

Install Elastic Cloud Enterprise on the first host to start a new installation, using a single combined cURL and bash command:

bash <(curl -fsSL https://download.elastic.co/cloud/elastic-cloud-enterprise-installer.sh)

Install Elastic Cloud Enterprise on additional hosts by including the --coordinator-host parameter to add them to an existing installation, using a single combined cURL and bash command. No availability zone is specified, so the installer will assign a default zone.

bash <(curl -fsSL https://download.elastic.co/cloud/elastic-cloud-enterprise-installer.sh) --coordinator-host 192.168.50.10"

Install Elastic Cloud Enterprise on the first host to start a new installation with additional parameters to set the runner ID, the host IP address, an availability zone, and capacity:

bash <(curl -fsSL https://download.elastic.co/cloud/elastic-cloud-enterprise-installer.sh) --runner-id runner-1 --host-ip 10.142.0.4 --availability-zone region-1a --capacity 30000"

Download the script and run it separately to start a new Elastic Cloud Enterprise installation:

curl -L -O https://download.elastic.co/cloud/elastic-cloud-enterprise-installer.sh
bash elastic-cloud-enterprise-installer.sh

Install Elastic Cloud Enterprise on nine hosts, placing them into three different availability zones. These steps require that you first download the installation script with curl -L -O https://download.elastic.co/cloud/elastic-cloud-enterprise-installer.sh on each host or make it available somewhere on your network.

  1. Start your installation by installing Elastic Cloud Enterprise on your first host and specify the runner ID and availability zone: 

    bash elastic-cloud-enterprise-installer.sh --runner-id runner-10.142.0.1 --availability-zone region-1a

  2. Add eight more hosts to the installation and specify a runner ID and availability zone, so that you end up with three hosts in each of the availability zones:

    1. Add two more hosts to the existing availability-zone region-1a: 

      bash elastic-cloud-enterprise-installer.sh --runner-id runner-10.142.0.2 --coordinator-host 10.142.0.4 --availability-zone region-1a
bash elastic-cloud-enterprise-installer.sh --runner-id runner-10.142.0.3 --coordinator-host 10.142.0.4 --availability-zone region-1b

    2. Add three hosts to a new availability-zone region-1b: 

      bash elastic-cloud-enterprise-installer.sh --runner-id runner-10.142.0.4 --coordinator-host 10.142.0.4 --availability-zone region-1b
bash elastic-cloud-enterprise-installer.sh --runner-id runner-10.142.0.5 --coordinator-host 10.142.0.4 --availability-zone region-1b
bash elastic-cloud-enterprise-installer.sh --runner-id runner-10.142.0.6 --coordinator-host 10.142.0.4 --availability-zone region-1b

    3. Add three hosts to a new availability-zone region-1c: 

      bash elastic-cloud-enterprise-installer.sh --runner-id runner-10.142.0.7 --coordinator-host 10.142.0.4 --availability-zone region-1c
bash elastic-cloud-enterprise-installer.sh --runner-id runner-10.142.0.8 --coordinator-host 10.142.0.4 --availability-zone region-1c
bash elastic-cloud-enterprise-installer.sh --runner-id runner-10.142.0.9 --coordinator-host 10.142.0.4 --availability-zone region-1c
  3. After installation completes, assign roles to the new runners in the Cloud UI.

Specify a custom JVM heap size of 1 GB for the admin-console service and 512 MB for the proxy service when you install Elastic Cloud Enterprise on the first host:

bash elastic-cloud-enterprise-installer.sh  --memory-settings '{"admin-console":{"xmx":"1024M","xms":"1024M"},"proxy":{"xmx":"512M","xms":"512M"}}'

Specify custom JVM settings that assign 80% of memory to the admin-console service and set heap size to 512 MB for the proxy service when you install Elastic Cloud Enterprise on the first host.

bash elastic-cloud-enterprise-installer.sh --memory-settings '{"admin-console":{"percentage":"80"},"proxy":{"xmx":"512M","xms":"512M"}}'
